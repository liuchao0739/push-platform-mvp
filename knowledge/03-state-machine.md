# 核心状态机

## 消息状态流转

```
业务方 POST /api/v1/push/send
        │
        ▼
    ┌─────────┐
    │ pending │  消息已入库，等待投递
    └────┬────┘
         │
    ┌────▼────┐
    │  sent   │  已发送（通过 WS 写入设备通道）
    └────┬────┘
         │
    ┌────▼────────┐
    │ delivered   │  客户端已确认收到（收到 ACK 回执）
    └─────────────┘

    任意阶段出错：
    ┌─────────┐
    │ failed  │  投递失败（WS 断线、ACK 超时等）
    └─────────┘
```

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| pending | 等待投递 | 消息入库时的初始状态 |
| sent | 已发送 | 服务端 WritePump 成功写入 WebSocket |
| delivered | 已送达 | 客户端回 `{"type":"ack","msg_id":"xxx"}` |
| failed | 失败 | 写入 WS 失败 / ACK 超时 / 设备永久离线 |

### 状态查询
- `pending` 的消息在消息队列中等待消费
- `sent` 但未 `delivered` 的消息可能存在 ACK 丢失（网络波动）
- 当前 MVP 版本未实现 ACK 超时重试，后续版本需补充

## 设备连接状态

```
    ┌──────────────┐
    │ disconnected │  初始状态 / 主动断开
    └──────┬───────┘
           │ connect()
    ┌──────▼───────┐
    │  connecting  │  正在建立 WebSocket 连接
    └──────┬───────┘
           │ 连接成功
    ┌──────▼───────┐
    │  connected   │  已连接，正常心跳中
    └──────┬───────┘
           │ 断线
    ┌──────▼────────┐
    │ reconnecting  │  正在重连（指数退避）
    └───────┬───────┘
            │ 重连成功 → connected
            │ 重连失败 → reconnecting（继续退避）
```

### 重连策略（指数退避）
| 重试次数 | 等待时间 | 累计时间 |
|----------|----------|----------|
| 1 | 1s | 1s |
| 2 | 2s | 3s |
| 3 | 4s | 7s |
| 4 | 8s | 15s |
| 5 | 16s | 31s |
| 6+ | 30s (上限) | 每次+30s |

## 设备注册状态

```
设备首次启动 → 生成 token → POST /api/v1/device/register
                                    │
                          ┌─────────▼─────────┐
                          │ status: "active"   │
                          └─────────┬─────────┘
                                    │
                          ┌─────────▼──────────┐
                          │ status: "inactive" │  设备卸载/长期离线
                          └────────────────────┘
```

- `active`：设备正常，可接收推送
- `inactive`：设备离线超过 7 天，不再投递（当前 MVP 未实现自动标记，预留字段）
